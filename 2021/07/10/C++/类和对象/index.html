类和对象

发表于2021-07-10|更新于2021-07-18
|阅读量:

类的封装

​ 类的基本思想就是数据抽象和封装，数据抽象是一种依赖于接口(interface)和实现(implementation)分离的编程技术。类的接口包括类所能执行的操作；而类的实现则包括类的数据成员、负责接口实现的函数体以及定义类所需的各种私有函数。

​ 封装让类的接口和实现分离开来，隐藏了它的实现细节，即用户只能使用接口而不能访问类的实现部分。

C++的封装就是把客观事物封装成抽象的类，有两层含义：把属性和方法进行封装，以及对属性和方法进行访问控制。

C++中类的封装：

​ 成员变量：C++中用于表示类属性的变量；

​ 成员函数：C++中用于表示类行为的变量；

类成员的访问控制

​ 在C++中可以给成员变量和成员函数定义访问级别：

  • public：成员和成员函数在类的内部外部皆能被访问；
  • private：成员和成员函数只能在类的内部被访问；
  • protected：只能在类的内部被访问，不能在类外被访问，用在继承里可以被访问；

注：strcut定义类时所有成员默认属性为public，class所有成员默认属性为private。

this指针

​ 在 C++ 中，每一个对象都能通过 this 指针来访问自己的地址。this 指针是所有成员函数的隐含参数。因此，在成员函数内部，它可以用来指向调用对象。

当你进入一个房子后，你可以看见桌子、椅子、地板等，但是房子你是看不到全貌了。对于一个类的实例来说，你可以看到它的成员函数、成员变量，但是实例本身呢？this是一个指针，它时时刻刻指向你这个实例本身。

  • ​ this只能在成员函数中使用，成员函数默认第一个参数为T const register this*。

  • ​ this指针不能在静态函数中使用。

静态函数如同静态变量一样，他不属于具体的哪一个对象，静态函数表示了整个类范围意义上的信息，而this指针却实实在在的对应一个对象，所以this指针不能被静态函数使用。

  • this指针在成员函数的开始执行前构造的，在成员的执行结束后清除；

  • this指针只有在成员函数中才有意义。

函数返回元素和返回引用

Test& add(Test &t2) //*this 函数返回引用
{
this->a = this->a + t2.getA();
this->b = this->b + t2.getB();
return *this; // *操作让this指针回到元素状态

}

注：返回一个引用相当于返回元素自身，程序中相当于返回对象t1本身，this实际上就是&t1，this实际上就是 &t1，返回值是 (*this)， 即返回 *(&t1) ,就是t1。

构造函数和析构函数

​ C++中的类可以定义与类名相同的特殊成员函数，这种与类名相同的成员函数叫做构造函数。构造函数在定义时可以有参数，可以没有任何返回类型的声明。构造函数一般情况下都是C++编译器自动调用，在一些情况下需要手动。

​ C++中的类可以定义一个特殊的成员函数清理对象，这个特殊的成员函数叫做析构函数。析构函数没有参数也没有任何返回类型的声明，析构函数在对象销毁时自动被调用。

​ 构造函数进一步细分成：无参构造函数、有参构造函数、拷贝构造函数。

拷贝(赋值)构造函数

​ 对象在调用拷贝构造函数时有4种调用时机：

  • 场景一：A a(b)；
  • 场景二：A a = b;
  • 场景三：形参是一个元素，实参传递给形参；
  • 场景四：函数返回值返回一个元素，匿名对象；

赋值的操作 和 初始化是两个不同的概念:

​ 初始化操作，也就是场景二会调用拷贝构造，但是简单的赋值操作不会调用拷贝构造。

​ 第四种场景关注函数返回值。函数g被调用时返回值是一个名字为A的Location对象，

匿名对象的去留

​ 拷贝构造中的场景四中，return的对象A产生的匿名对象会被析构释放掉，此时匿名对象是去，但是有一种情况是留。函数g()的返回值用一个变量m来接住，最后析构的是对象m，m对象其实就是由匿名对象抓转换而来，匿名对象转变成有名字的对象m。

拷贝构造结论

  1. 函数的返回值是一个元素 (复杂类型的), 返回的是一个新的匿名对象(所以会调用匿名对象类的copy构造函数)；
  2. 如果用匿名对象初始化另外一个同类型的对象, 匿名对象转成有名对象；
  3. 如果用匿名对象赋值给 另外一个同类型的对象, 匿名对象被析构；
  4. 当类中没有定义构造函数时，编译器默认提供一个无参构造函数，并且其函数体为空；
  5. 当类中没有定义拷贝构造函数时，编译器默认提供一个默认拷贝构造函数，简单的进行成员变量的值复制(浅拷贝)；

拷贝构造规则研究

  • 当类中没有定义任何一个构造函数时，C++编译器会提供默认无参构造函数和默认拷贝构造函数（仅仅是变量的简答赋值：浅拷贝）。
  • 当类中定义了拷贝构造时，C++编译器就不会提供无参构造函数；同理如果定义了有参或者无参构造函数，C++编译器不会提供和默认无参构造函数。

深拷贝和浅拷贝

​ 之前提到拷贝构造函数中系统默认提供的拷贝构造是浅拷贝，只会进行简答的赋值操作。

多对象的构造和析构

​ 当一个类(B)中有一个类成员本身就是一个类A或者一个结构，而且这个成员只有一个带参数的构造函数，没有默认构造函数。这时要对这个类成员进行初始化就必须调用这个类成员A的带参构造函数，这需要通过初始化参数列表实现。另外，类成员中若有const修饰，必须在对象初始化的时候，也必须通过初始化列表进行初始化。

对象的动态创建和释放

​ C++提供了较简便而功能较强的运算符new和delete来取代malloc和free函数。

malloc和free是函数，但是new和delete是运算符，运算符的执行效率更高！虽然为了与C语言兼容，C++仍保留malloc和free函数，但建议用户不用malloc和free函数，而用new和delete运算符。

​ new和delete运算符使用的一般格式：

​ 在执行new运算时，如果内存量不足，无法开辟所需的内存空间，目前大多数C++编译系统都使new返回一个0指针值。只要检测返回值是否为0，就可判断分配内存是否成功。

  • new能执行类型的构造函数，delete操作符能执行类的析构函数。
  • malloc只会开辟空间，但是并不会调用对象的构造函数，但是new不但开辟空间，还会调用构造函数来完成对象的初始化；同理free也不会调用类的析构函数。(所以不可以混用)

静态成员

静态成员变量

  • 关键字static可以用于说明一个类的成员，静态成员提供了一个同类对象的共享机制；
  • 把一个类的成员说明为 static 时，这个类无论有多少个对象被创建，这些对象共享这个同一个static 成员；
  • 静态成员局部于类，它不是对象成员；

静态成员函数

  • 静态成员函数数冠以关键字static；

  • 静态成员函数提供不依赖于类数据结构的共同操作，它没有this指针；

  • 在类外调用静态成员函数用 “类名 :: ”作限定词，或通过对象调用；

  • 静态成员函数中不能调用普通成员属性或者**普通成员函数**；
