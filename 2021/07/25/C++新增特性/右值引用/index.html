右值引用

发表于2021-07-25|更新于2021-10-13
|阅读量:

前置知识

​ 首先思考一个问题，C++与Java的主要区别是什么？有人会说Java中没有指针，这种说法不完全正确，Java中只是没有明显的使用指针，但是实际上Java中所有对象都是指针。

​ Object obj = new Object()，obj看似是一个对象，实际上是指向一个对象的指针，当一切都是指针时，就没有必要 使用指针这个概念了。但是在C++中不一样：

​ 所以在C++中存在两种值，一种是对象类型的普通值，一种是指向对象的指针值。

​ C++很多复杂的概念都是因为这，很多知识点例如运算符重载，拷贝构造函数、引用、右值引用等都是在这个背景下。但是C++不可以只用指针，因为指针的使用很容易导致内存写飞，而java不一样，java强大的gc机制清理内存。

const引用处理

​ 分析下面的代码，在执行“Buffer buf = getBuffer()”时，函数getBuffer中已经构造了一个对象，赋值的时候又构造一个对象，将函数内的对象拷贝给了函数外的对象。实际上代码只是想创建一个对象，那么函数外的buf有点多余，多构造一个对象就多一份消耗！

​ 那么就要思考了，是否可以将一个函数中临时变量，即将一个将亡值不死亡，而是继续使用？在C++中让一个常引用指向一个将亡值，那么这个将亡值的生命周期就会被延长到和这个常量引用相同的长度。

右值引用

​ 在某些情况下，有些对象不会释放可以一直使用，但是有些情况有些对象在函数运行完毕后就释放了，既然这个对象就要释放了，我们是否可以将即将消亡的资源拿来使用，从而避免重新申请内存再重新构造，从而大大提升了性能。

​ 现在在对象Buffer的构造函数中申请内存，在析构函数中释放内存，此时又出现了新的问题。setBuffer中使用的对象是getBuffer中的将亡值，setBuffer的入参是”const Buffer &”，原则上是不可以修改这个对象内存中内容的！

​ 另外，在setBuffer中是无法判断入参是一个将亡值，还是一个正常不会释放的值。为了不出现错误，并不能对所有情况进行重用，为了解决这个问题从而出现了右值引用。

首先a + b调用一次构造函数和拷贝构造函数，a+b生成一个临时对象temp_ab，temp_ab和c做加法生成一个临时变量temp_abc，temp_abc和d做加法生成的结果会直接拷贝给最后的结果r。

​ 在每一次return临时变量时都会使用一次拷贝构造函数，有没有什么办法避免呢？这里就需要了解一下什么是移动构造函数。

实现

​ 之前使用拷贝构造是需要分配内存的，现在使用拷贝构造函数的地方（return）都使用移动构造函数，移动构造函数中并不申请内存。而是采用右值引用的方法将将亡值重用。从而节约了资源。

重载运算符中的右引用

​ 进一步思考，对象temp_ab、temp_abc的作用都是传递数据，temp_ab + c = temp_abc。temp_ab是临时变量，c是普通变量，temp_ab 和temp_abc使用完都释放了，有没有什么办法? 可以将运算符重载进行优化修改。

右值引用是一个左值

但是运算符重载函数中“Matrix res(a.row, a.col);”调用的是普通构造函数，但是实际上“res = a +b”是可以表示为“a= a + b”的，那么希望运算符重载函数可以进一步优化。

​ 来进一步思考，m是一个右值引用，一个指向将亡值的右值引用，但是m本身不是将亡值。m是一个类型为右值引用的左值！所以“Matrix res= a;”中res本质上还是一个左值，那么调用的就是拷贝构造。

std::move

​ 那么有没有什么办法，将一个a移动给res呢？标准库提供了一个函数可以将一个值变成右值 —— **std::move()**。本质上就是将a中数值给偷取出来给了res，此时的a实际上已经空了！

​ 所以move函数是有风险的，经过move之后值被移动了，在访问就会出现bug。

move含义

​ C++11中提供了std::move()来将左值转换为右值引用，从而方便的使用移动语义。move是将对象的状态或者所有权从一个对象转移到另一个对象，只是转移，没有内存拷贝。 c++中所有容器都实现了move语义，方便我们实现性能优化。

注意：move只会对形如内存、文件句柄等资源的成员的对象有效，如果是基本类型，如int、char[10]数组等，如果使用move，仍然会发生拷贝（因为没有对应的移动构造函数）。
