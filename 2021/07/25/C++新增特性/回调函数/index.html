C语言中的函数

发表于2021-07-25|更新于2021-08-01
|阅读量:

函数指针

​ C语言中函数拥有三要素：名称、参数、返回值。C语言的函数有自己特定的类型。函数指针是用于指向函数入口的指针，函数名就是函数体入口地址。

函数指针的定义方法：

(1) 通过函数类型定义函数指针： FuncType pointer;*

(2) 直接定义： type (*pointer)(parameter list);

​ pointer： 函数指针变量名

​ type: 指向函数的返回值类型

​ parameter list： 指向函数的参数类型列表

==> 通过typedef为函数类型重命名： typedef type name(parameter list)

函数指针做函数参数

​ 上面的案例中， 任务的实现者和任务的调用者可以相互分开的，主函数中调用Mainop2函数，输入的参数是任务的实现者。也就是调用者完全不需要关心函数Mainop2的入参，只需要传入指针即可，任务实现者后续修改代码也不会影响调用者，因为相互拆分开的。

现在这几个函数是在同一个文件当中，假如 ：

​ int libfun(int (*pDis)(int a, int b))

是一个库中的函数，就只有使用回调了，通过函数指针参数将外部函数地址传入来实现调用，函数 add 的代码作了修改，也不必改动库的代码，就可以正常实现调用便于程序的维护和升级。

​ 当把一个函数指针作为函数的参数，传递给一个被调用函数，被调用函数就可以通过这个指针调用外部的函数，这就形成了回调。

回调函数

​ 案例背景：公司要开发下一代国民App“明日油条”，一款主打解决国民早餐问题的App，为了加快开发进度，这款应用由A小组和B小组协同开发。其中有一个核心模块由A小组开发然后供B小组调用，这个核心模块被封装成了一个函数，叫做make_youtiao()。

​ 最简单的做法就是A小组调用B小组写函数之后继续执行完成后续流程：

​ 但是现实并非如此，make_youtiao()这个函数需要处理的数据非常庞大，假设有10000个，那么make_youtiao(10000)不会立刻返回，而是可能需要10分钟才执行完成并返回！那么此时A小组调用B小组的接口之后就阻塞了。在这10分钟内该线程不会被操作系统分配CPU，也就是说该线程得不到任何推进。这种一直等待直到另一个任务完成的模式叫做同步。

一种更好的做法是：A小组继续执行自己的任务，等B小组完成后通知A小组，这样两个小组都不需要互相等待，这种模式被称为异步。

同步回调

​ 现在已经知道了由于make_youtiao(10000)这个函数10分钟才能返回，那么一种更好的方法是让make_youtiao()这个函数知道制作完油条后该干什么。

​ 更好的调用make_youtiao的方式是这样的：“制作10000个油条，炸好后卖出去”，因此调用make_youtiao就变出这样了：

​ make_youtiao(10000, sell);

现在make_youtiao这个函数多了一个参数，除了指定制作油条的数量外还可以指定制作好后该干什么，第二个被make_youtiao这个函数调用的函数就叫回调，callback。

​ 这样带来的好处就是不需要监视着等待，只需要将make_youtiao这个函数执行完后该做的任务交代给make_youtiao这个函数了。

make_youtiao()函数的功能由B小组编写给A小组调用，但是sell的函数是由A小组编写定义并通过回调方式让B执行。为什么不让B小组直接实现sell函数然后执行呢？

原因是：这款APP是由AB两个小组联合开发，B小组在编写make_youtiao时怎么知道A小组如何使用这个模块，无法知道执行完make_youtiao的后序操作。除此之外，如果现在C小组也要调用这个make_youtiao接口，接着不是sell而是存储store，那么B小组还要写一个store函数。B小组任务严重饱和显然不合理。

所以最佳做法：调用make_youtiao自己定义后序操作，通过回调让B小组在制作完成后调用回调函数执行后序流程！

​ 编写make_youtiao的B小组完全可以通过回调函数将接下来该干什么交给调用方实现，B小组的同学只需要针对回调函数这一抽象概念进行编程就好了。

​ 异步回调中这个任务实际上被分成两个部分，第一个部分是A小组需要处理的调用make_youtiao之前的工作。第二个部分就是make_youtiao的内部工作以及sell的内容，这块是在其他线程、进程、甚至另外一个机器上处理的。这个第二部分的调用不在A的控制范围之内。

也就是说：回调函数的本质就是“只有我们才知道做些什么，但是我们并不清楚什么时候去做这些，只有其它模块才知道，因此我们必须把我们知道的封装成回调函数告诉其它模块”。

正式定义

​ 回调函数本质上还是一个函数，在计算机科学中，回调函数指的是一段以参数的形式传递给其他代码的可执行代码。对于一般的函数，函数的编写方是我们自己，调用方也是我们自己。

​ 但是回调函数不一样，虽然函数的编写方是自己，但是函数的调用方不是我们，而是我们引用的其他模块！也就是第三方库，我们调用第三方库中的函数，并把回调函数传递给第三方库，第三方库中的函数调用我们编写的回调函数。

之所以需要给第三方库指定回调函数，是因为第三方库的编写者并不清楚在某些特定节点，比如我们举的例子油条制作完成、接收到网络数据、文件读取完成等之后该做什么，这些只有库的使用方才知道，因此第三方库的编写者无法针对具体的实现来写代码，而只能对外提供一个回调函数，库的使用方来实现该函数，第三方库在特定的节点调用该回调函数就可以了。
