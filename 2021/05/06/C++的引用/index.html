C++引用(一)

发表于2021-05-06|更新于2021-08-01
|阅读量:

普通引用

​ 我们在定义变量时：“int a = 4”，其中变量名a实质上就是一段连续存储空间的别名，是一个标号。程序中通过变量来申请并命名内存空间，可以使用变量名来使用内存空间。那么对一个内存空间只能取一个变量名吗？显然不是。

​ 在C++中新增加了引用的概念，引用可以看作一个已定义变量的别名。

​ 引用的意义在于：引用作为其它变量的别名而存在，因此在一些场合可以代替指针；另外，引用相对于指针来说具有更好的可读性和实用性。

引用本质

​ 引用在单独定义时必须初始化，很像一个常量；且引用有地址。所以引用在C++中的内部实现是一个常指针

引用本质： Type& name <=========> Type * const name

​ C++编译器在编译过程中使用常指针作为引用的内部实现，因此引用所占用的空间大小与指针相同。

函数返回值为引用

​ 函数getAA返回的是临时变量X的引用，实际上就是X的副本。对于a2是一个引用，a2变量存放的是变量的地址，在访问a2时，编译器在得知a2是引用时会自动进行：*a2，但是此时X对应内存已经释放了，根据地址查不到原来的内存空间。

​ 结论：

当函数返回值为引用时，如果返回的是栈变量：

(1) 不能成为其它引用的初始值；

(2) 不能作为左值使用；

返回值是static变量成引用

​ 函数j1()返回的是静态变量a，返回的是变量数值，而函数j2返回的是引用，需要注意的事项：

两个调用函数分别是“返回变量本身”和“返回变量的数值”，两者改变不一样；程序中若写“j1() = 10”这是错误的！相当于“11 = 10”，不成立；但是“j2() = 20”是正确的，因为j2()返回的是引用，是变量本身。这也是结论中说的既可以为左值也可以为右值。

​ 结论：

若函数返回静态变量或者全局变量：

(1) 可以成为其它引用的初始值；

(2) 可作为右值使用，可以作为左值使用；

指针作为引用

​ 指针作为引用时含义是一样的，相当于myp是main函数中pT的别名。

const引用结论

  1. const & int e <====> const int * const e;
  2. 普通引用 相当于int * const e1;
  3. 当使用常量（字面量）对const引用进行初始化时，C++编译器会为常量值分配空间，并将引用名作为这段空间的别名;
  4. 使用字面量对const引用初始化后，将生成一个只读变量;
  • 在引用的使用中，单纯给某个变量取个别名是毫无意义的，引用的目的主要用于在函数参数传递中，解决大对象的传递效率和空间不如意的问题。
  • 用引用传递函数的参数，能保证参数传递中不产生副本，提高传递的效率，且通过const的使用，保证了引用传递的安全性。
  • 引用与指针的区别是，指针通过某个指针变量指向一个对象后，对它所指向的变量间接操作，程序中使用指针，程序的可读性差；而引用本身就是目标变量的别名，对引用的操作就是对目标变量的操作。

引用要点

  1. 普通引用在声明时必须用其他变量进程初始化

一个变量被初始化时，初始值会被拷贝到新建的对象中。但是在定义引用时，程序是将引用和初始值对象进行绑定。一旦该引用被绑定之后，无法令引用重新绑定到另外一个对象，因此引用必须初始化！

  1. 引用即别名

引用并非对象，只是一个已存在的对象所起的别名。如果为一个引用尽心赋值，实际上是将值赋给与引用绑定的对象上。就是因为引用不是一个对象，所以不能定义引用的引用！

​ 3. 引用只能绑定在对象上

引用的类型需要和与之绑定的对象严格匹配，只能绑定到对象上，而不能与字面值或者某个表达式的计算结果绑定。

  1. 引用作为函数参数声明时不进行初始化

当实参传给形参引用的时候，只不过是c++编译器帮我们程序员手工取了一个实参地址，传给了形参引用（常量指针）。

void Test(int & a, int & b);

函数的参数是引用，当调用时函数参数实际就是生成两个实际对象的别名，在函数中对别名进行赋值取值操作就是对原对象操作。
