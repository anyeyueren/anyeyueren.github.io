智能指针

发表于2021-10-17|更新于2021-10-19
|阅读量:

​ 在了解智能指针之前需要先知道什么是内存泄露，内存泄漏（Memory Leak）是指程序中已动态分配的堆内存由于某种原因程序未释放或无法释放，造成系统内存的浪费，导致程序运行速度减慢甚至系统崩溃等严重后果。

​ 所以在使用C++语言进行编程时，使用了new那么就必须记得使用delete对申请的内存进行释放。这是需要程序员手动操作的，既然是手动释放内存，那么就很容易出现忘记释放内存情况。

​ 对于一个对象而言，我们通常在构造函数的时候申请空间，而在析构函数（在 离开作用域时调用）的时候释放空间。C++11引入了智能指针的概念，使用了引用计数的概念，程序员不再关心手动释放内存。

智能指针

​ 智能指针是存储指向动态分配（堆）对象指针的类，用于生存期的控制，能够确保在离开指针所在作用域时，自动地销毁动态分配的对象，防止内存泄露。

​ 智能指针的核心实现技术是引用计数，每使用它一次，内部引用计数加1，每析构一次内部的引用计数减1，减为0时，删除所指向的堆内存。

智能指针和普通指针的区别在于智能指针实际上是对普通指针加了一层封装机制，这样的一层封装机制的目的是为了使得智能指针可以方便的管理一个对象的生命期。

智能指针实际上是类，只是使用起来的语法规则跟指针一样。

智能指针种类

​ C++11 中提供了三种智能指针，使用这些智能指针时需要引用头文件**<memory>**。

  • ​ 独占资源所有权的指针： std::unique_ptr<T>
  • ​ 共享资源所有权的指针： std::shared_ptr<T>
  • ​ 共享资源的观察者： std::weak_ptr<T>

注：弱引用的智能指针，它不共享指针，不能操作资源，是用来监视 shared_ptr 的。std::auto_ptr 已被废弃。

shared_ptr

​ shared_ptr（共享指针）是指多个智能指针可以同时管理同一块有效的内存，共享智能指针shared_ptr是一个模板类。

​ 共享智能指针对象初始化完毕之后就指向了要管理的那块堆内存，如果想要查看当前有多少个智能指针同时管理着这块内存可以使用共享智能指针提供的一个成员函数 use_count。

// 管理当前对象的 shared_ptr 实例数量，或若无被管理对象则为 0。
long use_count() const noexcept;

​ 如果智能指针被初始化了一块有效内存，那么这块内存的引用计数 + 1，如果智能指针没有被初始化或者被初始化为 nullptr 空指针，引用计数不会 + 1。另外，不要使用一个原始指针初始化多个shared_ptr。

拷贝/移动构造函数初始化

​ 当一个智能指针被初始化之后，就可以通过这个智能指针初始化其他新对象。在创建新对象的时候，对应的拷贝构造函数或者移动构造函数就被自动调用了。右值可以是将亡值，也可以是纯右值。

​ 如果使用拷贝的方式初始化共享智能指针对象，这两个对象会同时管理同一块堆内存，堆内存对应的引用计数也会增加；如果使用移动的方式初始智能指针对象，只是转让了内存的所有权，管理内存的对象并不会增加，因此内存的引用计数不会变化。

std::make_shared初始化

​ 通过 C++ 提供的 std::make_shared() 就可以完成内存对象的创建并将其初始化给智能指针，函数原型如下：

template< class T, class… Args >
shared_ptr make_shared( Args&&… args );

T： 模板参数的数据类型；

Args&&… args：要初始化的数据，如果是通过 make_shared 创建对象，需按照构造函数的参数列表指定；

reset方法初始化

​ 共享智能指针类提供的 std::shared_ptr::reset 方法函数原型如下。reset函数主要有两个功能：

  • 解除该智能指针对某块内存的管理;
  • 通过reset使得某个智能指针指向另外一块内存，相当于对智能指针重新初始化;

void reset() noexcept;

template< class Y >
void reset( Y* ptr );

template< class Y, class Deleter >

unique_ptr

​ unique_ptr 是一个独享所有权的智能指针，它提供了严格意义上的所有权：

  1. 拥有它指向的对象；
  2. 无法进行复制构造，无法进行复制赋值操作；（也就是说不可能存在两个unique_ptr指向同一个对象）
  3. 可以使用移动构造和移动赋值操作来交换所有权；
  4. 当智能指针本身生命周期结束时，会调用给定的删除器将其所指向的对象删除；
