操作系统(一)

发表于2021-10-11|更新于2022-03-23
|阅读量:

计算机硬件组成

​ 图中是一个简单的计算机系统，内存、CPU、磁盘以及IO设备就可以组成一个简单的计算机。如果现在现在想在显示器上输出一个“hello”，CPU先发送指令给内存(假设是300)，想要将300中显示内容的数据发送到显存地址777上。即使用mov指令将内存300中内容发送到显存777上。如果一个简单的显示操作都如此繁琐的处理，会降低使用效率。

​ 计算机硬件实际上就是一个裸机，我们在开发时打印回显实际上只需要使用printf(“hello”)就能实现了，这就是因为使用了操作系统。操作系统也是软件，包在计算机裸机上，从而使得使用计算机更加方便。

计算机发展演变

图灵机

​ 计算机说到底就是一个计算模型，图灵的基本思想是用机器来模拟人们用纸笔进行数学运算的过程，可以把这样的过程看做是两个简单的动作：

​ 一是在纸上写上或者擦除某个符号；

​ 二是把注意力从纸的一个位置移动到另一个位置；

​ 例如现在要执行3 + 2的计算，人从纸上读取到3 + 2的信息，在脑子中计算得到等于5后写到纸上。用控制器来模拟人的大脑，读写 头模拟眼睛和笔。将3,2,+读取到控制器中得到5，写入到纸带上。

通用图灵机

​ 对于任意一个图灵机，因为它的描述是有限的。这种图灵机可以设置控制器动作、状态等等，功能不再是单一的。控制器动作时加法就是加法图灵机，是减法就是减法图灵机。

​ 从这里看通用图灵机已经非像我们所使用的计算机了。将不同的程序载入从而执行不同程序内容，实现不同的功能。

冯诺依曼存储程序思想

​ 冯诺依曼提出了存储程序的思想：将程序和数据存放到计算机内部的存储器中，计算机在程序的控制下一步一步处理。计算机由五大部件组成：输入设备、输出设备、存储器、运算器和控制器。

​ X86型CPU的指令指针寄存器IP相当于ARM型CPU中的程序计数器PC，用于控制程序中指令的执行顺序。

正常运行时，IP中含有下一条指令的偏移地址，一般情况下，每个内存中存取一次指令，IP就自动加一，从而保证指令的顺序执行。IP实际上是指令机器码存放内存单元的地址指针，IP的内容可以被转移类指令（如JMP）强迫改写，以改变程序执行的顺序。

注意：我们编制的程序不能直接访问IP，即不能用指令去取出IP的值或给IP设定值（如不能用mov指令给IP赋值）。

​ 计算机运作的过程是这样的：将程序载入到内存当中后，然后用一个指针(IP指针或PC指针)指向当前处理的指令，将该指令载入进指令寄存器(IR)来解释执行，就是控制器做的事情。

寄存器

寄存器 功能
程序计数器(PC, Program counter) 用于存放指令的地址;
指令寄存器(IR，Instruction Register) 用来保存当前正在执行的一条指令;
通用寄存器(GR，General register) 用寄存器可用于传送和暂存数据，也可参与算术逻辑运算，并保存运算结果；除此之外，它们还各自具有一些特殊功能;
累加器(AX) 用累加器进行的操作可能需要更少时间。累加器可用于乘、除、输入/输出等操作;
基地址寄存器(BX) 它可作为存储器指针来使用;
计数寄存器(CX) 在循环和字符串操作时，要用它来控制循环次数；在位操作中，当移多位时，要用CL来指明移位的位数;
数据寄存器(DX) 在进行乘、除运算时，它可作为默认的操作数参与运算，也可用于存放I/O的端口地址;
基址指针寄存器(BP,EBP) 主要用于存放堆栈内存储单元的偏移量，用它们可实现多种存储器操作数的寻址方式，为以不同的地址形式访问存储单元提供方便;
堆栈指针寄存器(SP,ESP) 寄存器SP称为堆栈指针寄存器;
源变址寄存器(SI) 变址寄存器主要用于存放存储单元在段内的偏移量，用它们可实现多种存储器操作数的寻址方式，为以不同的地址形式访问存储单元提供方便;
目的变址寄存器(DI) 变址寄存器不可分割成8位寄存器。作为通用寄存器，也可存储算术逻辑运算的操作数和运算结果;

PC寄存器：

​ 为了保证程序(在操作系统中理解为进程)能够连续地执行下去，CPU必须具有某些手段来确定下一条指令的地址。当执行一条指令时，首先需要根据PC中存放的指令地址，将指令由内存取到指令寄存器中，此过程称，为“取指令”。与此同时，PC中的地址或自动加1或由转移指针给出下一条指令的地址。

​ 此后经过分析指令，执行指令。完成第一条指令的执行，而后根据PC取出第二条指令的地址，如此循环，执行每一条指令。

IR寄存器：

​ 临时放置从内存里面取得的程序指令的寄存器，用于存放当前从主存储器读出的正在执行的一条指令。当执行一条指令时，先把它从内存取到数据寄存器（DR，Data Register）中，然后再传送至IR。

16位cpu通用寄存器共有8个：AX（累加器(Accumulator Register)）,BX（基地址寄存器(Base Register)）,CX（计数寄存器(Count Register)）,DX（数据寄存器(Data Register)）,BP（基址指针寄存器（Base Pointer））,SP（堆栈指针寄存器（Stack Pointer））,SI（源变址寄存器 （Source Index））,DI（目的变址寄存器（Destination Index））.八个寄存器都可以作为普通的数据寄存器使用。

取址执行

​ 首先将程序放到内存(存储器)中，然后用一个 指针指向程序入口，然后不断的自动取址执行，计算机进行工作原理实际上就是取址执行的过程。

操作系统的启动

​ 当我们开启电脑的开机键之后，正在启动计算机，这时候操作系统在上电初始化。之前说了计算机运行实际上就是取址执行的过程，此时应该关注启动时指针IP所指向的内容！

​ 那么当打开电源之后，计算机执行的第一句指令是什么？PC = ？这就要分析下硬件系统了，以X86的PC为例，上电时有一部分内容是固化的 — ROM BIOS映射区（0xFFFF0这个地址处）。

  1. X86 PC刚开机时CPU处于实模式；

  2. 开机时，CS = 0xFFFF，IP = 0x0000;

    实模式的寻址CS:IP（CS左移4位 + IP），和保护模式不一样！

  3. 寻址0xFFFF0，即找到ROM BIOS映射区；

  4. 在这块映射区中的程序负责很多初始化的工作：检查RAM、键盘、显示器、软硬磁盘等；

  5. 将磁盘的0磁道0扇区（引导扇区，操作系统第一个扇区）读入512字节，到0x7c00处，将数据读入到内存中；

  6. 设置cs = 0x07c0， ip = 0x0000；

引导扇区

​ 引导扇区就是启动设备的第一个扇区，将引导扇区的512个字节读入到内存中。开机时按住del键可以进入启动设备设置界面，可以设置为光盘启动！

​ 操作系统所有的故事都是从这里开始的，因此硬盘的第一个扇区上存放着开机后执行的第一段我们可以控制的程序。引导扇区中是一段汇编代码：bootsect.s。

系统启动

​ 操作系统装在了硬盘上(磁盘)，而计算机原理是取址执行，这就要将代码放在内存中才能取址，再执行完成工作。所以启动后第一步就是将磁盘中操作系统读取到内存中。

​ 开机后，第一段代码在磁盘的引导扇区中，即bootset.s中代码来完成工作。在读取bootset.s代码时是分段读取的，首先读取setup，然后在屏幕上打印出来一个logo，接下来继续取址执行交给下一步操作，就是setup.s。

​ setup模块就是setup.s，根据名字可以想到：setup将完成OS启动前的设置。

内存地址 长度 名称
0x90000 2 光标位置
0x90002 2 扩展内存数
0x9000C 2 显卡参数
0x901FC 2 根设备号

​ 所以setup的主要工作就是：挪动了一下操作系统，将操作系统从0地址开始处排布，然后获取了硬件参数，为了将来操作系统能够成功建立起来做好准备。setup做完工作后就退出舞台了，但是操作系统需要继续向下执行，setup最后又做了这样一件事：进入保护模式。

保护模式

​ 在setup.s中最后执行了下面这一行指令，按照之前的做法“jmpi 0,8 ”中ip为0，cs为8，cs左移四位加ip得到地址0x80，这个地址在system模块，真正跳转过去就会死机！这里真正希望的不是跳到80，而是想跳到0地址处。

​ 在这里需要转变的是寻址方式，我们知道CS是一个16位寄存器，IP也是一个16位寄存器，那么CS左移四位加上IP也只能得到一个20位的地址。20位地址最多只能达到1M，那么计算机能访问的内存大小只有1M，这样的计算机肯定是不合适的。现在计算机要访问的4G的物理内存，那么要如何才能实现？可以改变寻址方式。

​ 那么要切换的能访问4G的寻址模式，从16位模式切换到32位模式，也就是从实模式切换到保护模式。16位和32位主要区别在于寻址不一样，说到底就是对于程序的解释是不一样的。保护模式不能再用cs左移四位加ip的方式寻址了。现在需要切换到新的电路。

​ 下图便是一个CR0寄存器，x86_32的CR0为32bit。X86_64下为64bit，其中低32bit与x86_32的CR0保持一致，高32bit没有定义，作保留使用，除了bit 4其他所有位都是可读可写的。

  • PE位：（Protected-Mode Enabled，保护模式使能）PE等于0时，表示CPU处于实模式，PE等于1时，表示处于保护模式，并使用分段机制；
  • PG位：（Paging Enable，分页使能）该位控制分页机制，PG=1，启动分页机制；PG=0,不使用分页机制；

保护模式下地址翻译

​ 保护模式下寻址首先需要知道一个GDT（全局表述表）。给出一个逻辑地址：

  • 首先根据TI是0还是1确定是要转换GDT中的段还是LDT中的段。gdtr寄存器中可以得到GDT的入口地址，在ldtr寄存器中得到LDT的入口地址。这里先找到gdtr寄存器，从中得到GDT的基址；
  • 有了GDT的基址，又有段寄存器中保存的索引，可以得到段寄存器“所指”的那个表项，既所指的那个描述符；
  • 得到了描述符，就可以从描述符中得到该描述符所描述的那个段的起始地址；
  • 有了段的起始地址，将偏移地址拿过来与之相加，便能得到最后的线性地址；
  • 有了线性地址（虚拟地址），经过变换，即可得到相应的物理内存地址；

​ 所以实模式下是cs左移4 + ip，而保护模式下是根据cs（选择子）查找 + ip进而获取到物理内存地址。保护模式下的地址翻译使用硬件来完成的，最大的好处就是速度很快！

​ 既然jmpi 0,8中cs为8，也就是说通过cs为8去gdt表中查找，对应的数据。所以setup.s要初始化gdt表。

​ gdt表在x86架构中用来存储内存的分段信息，通过段选择子进行访问，表的大小=0x10000=65536字节，每个表项占8字节，第一个表项为空，不使用，因此一共有8191个可用表项。表项结构如下：

​ 将0x07FF, 0x0000, 0x9A00, 0x00C0按个填入得到0x00C09A00000007FFF，将段基地址分散的各段拼接起来得到0x0000，所以得到地址0，那么setup.s最后跳到0地址处执行，跳到了操作系统的核心地方。

​ 再来看一下setup.s做的事情：

  • 读取了一些硬件参数，比如空间大小，为后面建立操作系统打下基础；
  • 将操作系统挪到0地址处，操作系统核心代码加载到内存后一直存放在那里；
  • 从实模式切换到保护模式；
  • 最后用了一条高级的32位汇编指令跳转到0地址处；

​ 在setup.s完成后会进一步跳到system模块执行。

system模块

​ 从setup.s执行完毕后就跳到了system模块执行，system是由一堆代码编译出来的。system模块(目标代码)中的第一部分代码? head.s 。上面的“jmpi 0,8”就是跳到head.s去执行。

​ 注意从head.s跳转到main.c，是从一个汇编代码跳转到main.c，本质上与从c跳c没有区别。

​ 下面是head.s的内容，setup是进入保护模式，head是进入之后的初始化。setup中创建了GDT表，在head中还会进行初始化。head.s中都是32位汇编代码。

​ setup_paging执行ret后，按照压栈顺序，那么出栈的顺序第一个是main，所以执行main函数。而三个压栈的0实际上是main函数的三个参数。main函数返回时进入L6，死循环。

main.c

这里就是从一个汇编代码调用了一个c语言代码。main函数的工作就是初始化：内存、中断、设备、时钟、CPU等内容的初始化。

小结

​ 本节主要讲了在操作系统启动时期所做的事情，讲了bootsect.s，setup.s，head.s，main.c。最后实现的样子是将操作系统的代码读入到内存的0地址处，创建并初始化了一些初始的数据结构，比如GDT表、IDT表、mem_map等等。
