完美转发

发表于2021-10-13|更新于2021-10-13
|阅读量:

​ 在看完美转发之前，先要熟悉右值引用以及move的用法。

左值与右值

​ C++表达式划分为左值和右值，注意通常左值和右值都是针对表达式的！

认识左右值

左值是在内存的表达式，能够用内置的&进行取地址，其它可以算右值;

右值更多的是一种“值”的表达，其中广义上右值又包含了纯右值与亡值，亡值一般就是我们说的生命周期即将结束的表达式，上面的TestClassA(100)是一个典型代表;

​ 常被错认成右值的左值：

  1. 字符串字面量，如："Hello";
  2. 内置的前++与前–，如：++a;
  3. 变量类型是右值引用的表达式，如：TestClassA&& ra = TestClassA(1000);,ra这里是左值;
  4. 转型为左值引用的表达式，如：static_cast<double&>(fValue);
  5. 内置*解引用的表达式，如：*pkValue;

​ 常被错认成左值的右值：

1.非字符串的字面量以及枚举项，如：nullptr,true;

2.**内置的后++与后–**，如：a--;

3.内置的算术，逻辑，比较表达式，如：a+b，a&b，a||b, a<b;

4.内置取地址表达式，this指针，如:&a;

5.lamda表达式,如：[](int a){ return 2*a; };

6.转型为非引用的表达式，如：static_cast<double>(fValue), (float)42;

7.转型为右值引用的表达式，如：static_cast<double&&>(fValue)，std::move(x);

左值引用和右值引用

​ 引用是变量的别名，必须进行初始化操作。引用操作从反汇编层面看就是指针。正常情况下左右引用只能处理对应的左右值，不能随意配对。

指向左值的引用就是左引用，我们单个&来表示，C++11前一直使用的;对右值的引用是右引用，我们用&&来表示。

特别的const T&

​ 有没有直接对右值进行左引用的？还真有，那就是**const T&**。常量左引用既能接受左值，也能接受右值。当接受右值时，就是使得右值的生命周期被延续。

​ 如果C++中，函数传参，不改变参数时，尤其是大数据，尽量使用const T&。

一个类的拷贝构造常使用**const T&**来描述入参，

当传入的是一个右值，函数的形参是const T&，将亡值被const T&延长了生命周期，此处并不会调用T的拷贝构造；

当传入的是一个左值，函数的形参是const T&，会调用T的拷贝构造函数；

左值转右值引用

​ 引用实质就是操作地址指针，而右值不能取地址，理论上无法进行左引用。一个左值可以取地址，无法直接进行右引用，但是可以通过move间接进行右引用。

移动语义

​ 移动的语意类似于一种资源托管，资源从一个对象安全的交给另一个对象进行管理，且保证安全。通俗理解就是一些带资源的对象需要拷贝时，想要有浅拷贝的效率，还想要深拷贝析构时安全的效果，正所谓鱼和熊掌不可兼得，那么被拷贝者资源权转给拷贝者，自己不再使用，来保证安全和高效。

深拷贝与浅拷贝

​ 我们都知道两个对象之间如果不编写拷贝构造，那么对象之间的拷贝就是简答的数值传递，这就是浅拷贝。当一个对象在构造是需要new一块内存，析构的时候释放一块内存，浅拷贝后就是两个对象对应同一块内存，那么在对象释放时必定造成重复释放的问题！那么这时就必须使用深拷贝来解决。

​ 但是深拷贝需要频繁的申请释放资源，如果申请的内存很大，有些变量只是短暂使用的临时变量，很快就要释放资源析构了。那么这样做的代价又太高。

移动的引入

​ 一个对象的资源在销毁前，我们将其转移给其它对象再用起来，这样能减少资源带来的构造开销，程序获得更高的效能。C++11增加&&右值引用，简事情变简单:

  • 想要深拷贝的就T(const T&)实现;

  • 想要移动拷贝的就实现T(T&&);

两者都实现时，一旦传入是个右值，就优先触发移动拷贝构造调用，使用移动语意，转移资源，减少拷贝。

​ 注意：只有下面4个函数同时没有手动定义时，编译器才会帮我们生成移动拷贝构造函数，原型为：**T(const T& )**。（拷贝构造，拷贝赋值运算符，移动赋值运算符，析构函数）

引用折叠

​ 之前提到std::move可以将一个左值转变为右值。

​ 在分析代码之前先来看一个规则：C++的引用折叠机制。

  • T& & 折叠成 T&
  • T& && 折叠成 T&
  • T&& & 折叠成 T&
  • T&& && 折叠成 T&&

​ 如果入参__t是一个左引用，强转会添加&&右引用，那么最后经过折叠（第二条）之后就变成了左引用，这显然不是想要的结果。一旦用remove_reference去掉引用后，我们再加上&&，就能明确是右引用了！里面函数体也很简单，就是直接static_cast强转，并不复杂。进而可以让我们后绪触发移动语意的相关函数，比如移动构造，移动赋值构造。

​ move主要针对本身是左值的表达式转为右引用，如果本身是右值，加上也没有什么意义。

万能引用

​ 案例中：ra2222是右值TestClassA(2222)的引用，但ra2222这个表达式本身是个左值，同理，参数是右值引用的函数中内部表达式rA实际也已经成了左值，所以再调用一次就打印了：left Value 4444。

​ 问题来了，如果函数体内还想保持右值引用，怎么办呢？可以使用move函数来左切右，但是有时候：函数希望传进来的是左引用，函数体内也保持左引用；传进来的是右引用，函数体内也保持右引用！

​ 此时就需要使用模板函数的万能引用和完美转发实现。

万能引用语法

​ 注意下面事项：

  • 必须是函数模板，可以模板参数是单个，也可以是多个模板参数，形式为T&&;
  • 万能引用可以接受左值，也可以接受右值，而不是上一个案例中必须左右引用写两个函数对应；
  • 万能引用的T不能被再修饰，否则转为普通右值引用；
  • 如果想在模板类中的模板函数使用万能引用，不能使用模板类的参数，否则转为普通右值引用；

完美转发

​ 有了前面的万能引用，实现了接受左右引用，std::forward()来了，就是因为能完美正确的保持原来的左右引用关系并向下层次转化，不再因右引用表达式是左值从而变掉，所以叫完美转发。
