容器vector(一)

发表于2021-08-08|更新于2021-08-09
|阅读量:

​ 一个容器就是一些特定类型对象的集合，顺序容器为程序提供了控制元素存储和访问顺序的能力，这种顺序不依赖于元素的数值，而是与元素加入容器时的位置相对应。

​ vector是可变大小数组，支持快速随机访问，在尾部之外的位置插入或者删除元素会比较慢。相对应的array容器是固定大小数组，支持快速随机访问，但是不能删除和添加元素。之前接触到的string与vector相似，但是是专门用于保存字符的。本章将注重讲解vector容器。

除非有更好的理由，不然vector是最好的选择。

vector简介

​ vector是表示可变大小数组的序列容器， 就像数组一样，vector也采用的连续存储空间来存储元素。也就是意味着可以采用下标对vector的元素进行访问，和数组一样高效。但是又不像数组，它的大小是可以动态改变的，而且它的大小会被容器自动处理。

vector使用方法

vector声明和初始化

vector容量

  • 向量大小： vec.size();
  • 向量最大容量： vec.max_size();
  • 更改向量大小： vec.resize();
  • 向量真实大小： vec.capacity();
  • 向量判空： vec.empty();
  • 减少向量大小到满足元素所占存储空间的大小： vec.shrink_to_fit();

数据操作

  • 多个元素赋值： vec.assign(); //类似于初始化时用数组进行赋值

  • 末尾添加元素： vec.push_back();

  • 末尾删除元素： vec.pop_back();

  • 任意位置插入元素： vec.insert();

    1. 在指定位置loc前插入值为val的元素,返回指向这个元素的迭代器:

    ​ iterator insert( iterator loc, const TYPE &val );

    1. 在指定位置loc前插入num个值为val的元素：

    ​ void insert( iterator loc, size_type num, const TYPE &val );

    1. 在指定位置loc前插入区间[start, end)的所有元素：

    ​ void insert( iterator loc, input_iterator start, input_iterator end );

  • 任意位置删除元素： vec.erase();

    1. 删除指定位置的元素：
     ​               **iterator erase ( iterator position );**
    1. 删除一段区间的元素：
     ​               **iterator erase ( iterator first, iterator last );**

    注：返回值是一个迭代器，指向删除元素的后一个元素，如果删除的是最后一个元素，那么返回值就是end()。

  • 交换两个向量的元素： vec.swap();

    ​ 交换两个同类型的vector的内容，两个容器的size可以不同。

    ​ void swap ( vector<T,Allocator>& vec );

  • 清空向量元素： vec.clear();

    源码中就是调用erase函数进行删除，void clear() { erase(begin(), end()); }

  • 多元素赋值： vec.assign();

    ​ void assign ( InputIterator first, InputIterator last );

    ​ void assign ( size_type n, const T& u );

  • 元素翻转： vec.reverse();

    ​ void reserve ( size_type n );

vector扩容机制

​

​ 为了达到快速随机访问的效果，vector在设计时容器内部的元素以连续的方式存放。vector声明的时候是没有分配堆内存的，执行push_back()的时候动态增长的。那么如果在增长过程中这块连续的内存不够了会怎么办？

​ 当向一个存满的vector中添加数据时，vector的增长原理是这样的：

  • 发现空间不足时就重新申请一块更大的内存；
  • 将旧空间的数据拷贝到新的空间；
  • 最后将旧空间释放；

​ 从上面的运行可以看出vector的初始扩容效率低，需要频繁增长，不仅操作效率很低，而且频繁的向操作系统申请内存，这样容易造成过多的内存碎片，所以这个时候需要合理使用resize()和reserve()方法提高效率减少内存碎片。

resize()

​ 如果resize中所指定的n小于vector中当前的元素数量，则会删除vector中多于n的元素，使vector得大小变为n，但是容量capacity不会改变。

​ 如果所指定的n大于vector中当前的元素数量，小于capacity容量值，则会在vector当前的尾部插入适量的元素，使得vector的大小变为n。此时容量依然不会变。

​ 如果n比capacity都要大，那么会自动为vector重新分配空间，此时容量会变化。

reserve()

​ reserve方法被用来重新分配vector的容量大小：

  • 只有当所申请的容量大于vector的当前容量时才会重新为vector分配存储空间。新分配后的容量即reserve大小，size大小没有变，所以不会产生新的对象；

  • 小于当前容量则没有影响,size、capacity都没有影响；

    reserve方法对于vector元素大小没有任何影响，不创建对象。
